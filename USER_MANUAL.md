# 家庭智能药箱 (Family Pillbox) 使用与系统架构说明书

欢迎使用**家庭智能药箱 (Family Pillbox)** 应用程序。本说明书旨在提供从系统架构设计、数据库模型，到具体业务运作和真实用户交互流程的完整解析，帮助系统开发者、运维人员及家庭最终用户深入理解并高效使用本系统。

---

## 1. 软件应用定位与核心价值
在多代同堂的现代家庭中，药品的安全存储与规律服用至关重要。药品容易混淆、过期，或说明书丢失，导致在突发状况下难以快速找到正确的用法与配伍禁忌。

**家庭智能药箱**专为家庭场景设计，通过**离线全本地化存储、家庭多成员隔离验证、药盒存储位置图谱、说明书高清拍照/相册存根、双重用药行为追溯审计**等功能，提供一站式的家庭药品全生命周期追踪管理。

---

## 2. 软件系统架构设计 (System Architecture)

本系统完全基于 **MVVM (Model-View-ViewModel)** 经典架构模式设计，充分结合现代 Android 与 Jetpack 技术栈，确保高流畅、低能耗、无网络依赖的完美本地化隐私安全。

```
                    ┌───────────────────────────────┐
                    │      Jetpack Compose UI       │ (View 层)
                    └───────────────┬───────────────┘
                                    │ 观察 UI 状态 / 派发用户意图
                    ┌───────────────▼───────────────┐
                    │        MainViewModel          │ (ViewModel 层 - 业务逻辑)
                    └───────────────┬───────────────┘
                                    │ 调度底层响应式数据源
                    ┌───────────────▼───────────────┐
                    │        AppRepository          │ (Repository 层 - 统一数据门面)
                    └───────────────┬───────────────┘
                                    ├──────────────────────────────┐
                    ┌───────────────▼───────────────┐    ┌─────────▼─────────┐
                    │      AppDatabase (Room)       │    │   BackupHelper    │ (数据备份/恢复)
                    └───────────────┬───────────────┘    └───────────────────┘
                                    │ SQLite
                    ┌───────────────▼───────────────┐
                    │         本地数据表            │
                    │   (Users / Meds / Logs)      │
                    └───────────────────────────────┘
```

### 2.1 核心技术栈 (Technology Stack)
*   **开发语言**：Kotlin (1.9+) - 提供严格的空安全、类型推断与流畅的代码语法。
*   **界面层**：Jetpack Compose (Material Design 3 标准) - 声明式 UI 设计，组件原生硬件加速，完美结合动态配色与流畅手势。
*   **架构模式**：MVVM 搭配 Jetpack Lifecycle (ViewModel, LiveData/StateFlow)。
*   **数据存储**：Room Database (基于本地 SQLite) - 实现强类型、事务安全、高并发吞吐的本地结构化持久化。
*   **导航导航**：Compose Navigation - 优雅的单 Activity-多 Screen 路径切换。
*   **异步机制**：Kotlin 协程 (Coroutines) 与 响应式数据流 (Flow) - 保证主线程完全零阻塞、数据库读写流畅顺滑。

## 3. 本地数据库关系与物理图谱 (Data Domain Models)

本系统采用三大 Room 实体表构成完整的用药监督生态链：

```
                    ┌─────────────────┐
                    │      User       │  <-- 用户实体（多成员切换，支持头像全称）
                    ├─────────────────┤
                    │ - id (PK)       │
                    │ - username      │
                    │ - passwordHash  │
                    │ - fullName      │
                    └─────────────────┘
                             
                    ┌─────────────────┐
                    │   Medication    │  <-- 药品库存实体（储存详情、位置、说明书Base64）
                    ├─────────────────┤
                    │ - id (PK)       │
                    │ - name          │
                    │ - quantity      │
                    │ - unit          │
                    │ - efficacy      │
                    │ - dosage        │
                    │ - frequency     │
                    │ - timing        │
                    │ - form          │
                    │ - targetGroup   │
                    │ - expiryDate    │
                    │ - instructionsImage (String? Base64)
                    └─────────────────┘
                             │ (通过 medicationId 与实体软关联)
                             ▼
                    ┌─────────────────┐
                    │  MedicationLog  │  <-- 审计日志实体（记录入库、服用、销毁、修正）
                    ├─────────────────┤
                    │ - id (PK)       │
                    │ - medicationId  │
                    │ - medicationName│
                    │ - username      │
                    │ - actionType    │
                    │ - quantityChange│
                    │ - timestamp     │
                    └─────────────────┘
```

### 3.1 药品表 (Medication)
重点字段解析：
*   `expiryDate` (Long): 过期截止日的毫秒级时间戳。系统用来自动排序、置顶并提示临期或过期风险。
*   `instructionsImage` (String?): 采用高保真 Base64 字符串形式，将拍摄或选取说明书图片直接存于本地数据库。如此设计，即使手机清理相册、更换位置，说明书也随数据库一同绑定，永不丢失。

### 3.2 审计日志表 (MedicationLog)
记录每一次药品状态变更。包含操作人姓名、变更数量（减少用负数表示，增加用正数表示）、操作行为，自动生成精美的时间线历史（Timeline）。

## 4. 全系统功能板块解析 & 真实使用流程 (UI Flows)

进入应用后，全流程逻辑清晰，一览无余：

### 4.1 成员安全登录登录 (LoginScreen)
1.  **首登注册**：如果是新安装系统，可以使用快捷创建账户功能录入系统管理员及家庭成员基本信息。
2.  **安全登录**：通过安全验证隔离家庭成员权限，在后台记录用药、入库的人员名称。
3.  **多账户无缝切换**：确保长辈、子女均能在安全独立的记录下运行。

### 4.2 今日概览看板 (HomeScreen)
进入应用首屏，系统通过精制卡片和醒目的 Material 3 极简数据图章来展现全家药品的综合信息：
1.  **三大健康指标统计**：
    *   **药品总数**：当前库存中药品的类目与数量。
    *   **临近过期/已过期**：智能计算当前时间戳与 `expiryDate` 的对比。凡是已过期的药箱，用猩红色醒目警示；30天内临期的用橙黄色温暖提醒，杜绝误服过期失效药品的家庭悲剧。
    *   **今日用药日志统计**：展示今日全家人已服用的药品明细和记录数。
2.  **过期药品红色拦截横幅**：当存在已过期药品时，顶端会出现智能横幅提醒快速下架处置。
3.  **快捷记录清单**：可以直接在此一键增加用药日志，无需频繁切换。

### 4.3 智能药盒清单 & 服用说明书存储 (MedicinesScreen)
这是本系统的核心业务中枢。包含完善的药品列表检索、排序（按过期时间、药名等）、以及详实的信息卡片：
1.  **多维条件检索**：
    *   支持**关键字搜索**药品名称、功效（如“退烧”、“感冒”）。
    *   可通过**人群分类**筛选（“成人用药”、“儿童专用”、“全家通用”）。
    *   可通过**状态标签**过滤（“正常有效”、“临期警戒”、“已经过期”）。
2.  **新增及编辑药品 (带防抖与说明书存留)**：
    点击悬浮按钮 (FAB) 可调出极为精美的药品录入对话框：
    *   填入药名、库存量、包装单位、每次服用剂量、服用频率等。
    *   **分类剂型与用药人群选择**：采用 LazyRow 水平滚动药丸标签（Pills），单选轻松时尚。
    *   **说明书拍照 & 相册剪裁**：支持高阶的原生相机调用与图库选取。应用提供了高级的 `ActivityNotFoundException` 捕捉（即使极弱配置的定制系统上没有主相机也不会崩溃，会优雅Toast提示），拍摄的图片将自动通过高效算法压缩并存储。
3.  **高级删除防误触确认机制 (Double Confirmation)**：
    为了极致拟真、保证数据不丢失，点击右下角删除按钮时，**不再是一点即失**，而是展示一个详尽的**红色二级确认警告对话框**。
    *   对话框提醒用户：“删除后该药品的【所有库存数据】、【绑定的说明书图片】以及相关的【服药日志历史记录】都将被永久清除，且该操作不可撤销！”
    *   提供了反悔机制，用户可点击“考虑一下 (取消)”完美撤销，也可授权“确定删除”。

### 4.4 智能服药与操作审计日志 (LogsScreen)
家庭智能药箱提供非破坏性的库存变化记录。任何关于库存数量的变化均在日志页面被记录：
*   **记录产生逻辑**：
    *   新药品入库 -> 自动产生 `[入库]` 日志。
    *   日常服药 -> 控制面板点击已服药，扣减库存 -> 自动产生 `[用药]` 日志，带上有药人、消费药量。
    *   清理失效期药品 -> 底层执行安全剥离 -> 自动产生 `[过期处理]` 日志。
*   **交互界面**：
    *   包含清晰的垂直轴时间线历史。
    *   顶部拥有**一键清空日志**（管理员专属）或搜索单次事件功能。

### 4.5 数据备份备份与一键恢复 (SystemScreen)
为了让家庭在更换新设备、或者防备系统重置时依然保留全量家庭健康数据，本系统自研了一套**无网 JSON 数据极速备份恢复系统**：
1.  **全本底离线备份**：
    *   一键将所有的：用户信息、药品卡片、所有的服药说明书照片、用药详细日志，打包成一个格式高度规范、带有缩进的可读 JSON 字符串文件。
    *   用户可以点击一键复制，存至微信、便签、云盘中。
2.  **全本底一键恢复**：
    *   将旧设备的 JSON 备份文本拷贝进来，点击“解析并恢复备份”，系统将立刻在 SQLite 事务中完成数据的完整性恢复，药品照片瞬间找回！
3.  **多成员切换与清空**：
    *   提供便捷的用户退出，并可选一键初始化沙盒，保护全家隐私。

## 5. 关键技术特性设计亮点
1.  **零外网依赖**：所有药品说明书图片、用药日志在真机上均以二进制/Base64 在本地 Room 中高效流动。断网离线、山区户外、弱网环境下皆可 100% 顺畅使用，保障家庭急用。
2.  **极致视觉体验**：使用 Material 3 卡片边框设计，针对长辈优化了字体与常用剂型、位置标签的色彩区分，操作触感自带 Material Ripple 水波回馈。
3.  **双重防呆设计**：不仅在**删除药品**时提供致命危险提醒，还在**启动相机与相册**时对各类老旧、定制机型可能缺失内置相机应用的情况做了高阶适配：
    ```kotlin
    try {
        takePictureLauncher.launch(uri)
    } catch (e: ActivityNotFoundException) {
        Toast.makeText(context, "未找到相机应用，请通过相册选取图片！", Toast.LENGTH_LONG).show()
    }
    ```

## 6. 技术问答：关于 `IME_INSETS_HIDE_ANIMATION` 提示
*   **日志表现**：`FrameTracker: force finish cuj, time out: J<IME_INSETS_HIDE_ANIMATION::1...>`
*   **原理解析**：此提示并非系统崩溃或语法错误，属于 Android OS 系统层级的**流畅度运行调试预警**。
*   **触发机理**：当你在输入框输入药名或剂量后调起手机输入法（IME），点击空白处、点击确定或通过物理手势隐藏键盘时，Android 系统的 `FrameTracker` 会启动一个 CUJ (Critical User Journey) 动画帧监控器进行录制。由于虚拟机或部分真机渲染负荷（比如要一边做说明书转换，一边异步重排数据流），这个键盘隐藏动画在规定微妙数内没有显式将“已关闭信号”反馈给 Android 内核监控，导致监控组件触发超时并主动断开监测并输出此条警告日志。
*   **解决与表现说明**：系统已经在 UI 交互中充分使用了 Compose 最新的 `WindowInsets` 与 Edge-to-Edge 处理，此调试警告被 Android 底层静默回收，**对应用的实际用药业务、本地数据安全、及日常流畅度均 100% 没有任何负面影响**，用户可以完全放心使用。
